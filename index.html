HTML5 Introduction

    HTML5 Introduction

  • HTML5 is the latest revised version of HTML, which was accomplished by W3C in October,2014.
  • The purpose of the design of HTML5 is to support multi-media in mobile devices.
  • HTML5 is very easy to learn.

    What HTML5?

  • HTML, the last version of HTML4.01 waspublished in 1999.From then on the world of Web had greatly changed.
  • The purpose of the design of HTML5 is to support multi-media in mobile devices.
  • HTML5 is still in development.But most of mordern browsers have already supported HTML5.

    What is the beginning of HTML5?

  • HTML5 is the result of the cooperation of W3C and WHATWG(Web Hypertext Application Technology Working Group.
  • WHATWG dedicate to on web tables and applications,and W3C focus on XHTML2.0.In 2006,these two groups decided to work together to create a new version of HTML.

    • Some interesting features in HTML5:

  • Canvas which is used to paint.
  • Video and audio which are used to media playback.
  • Support more on local offline storage.
  • Brand new content-control element such as article,footer,header,nav,section.
  • New table and form controlling element such as calender,date,time,email,url,search.

Semantic Elements

There are lots of new semantic elements in HTML5

Tag Description
<article> define individual content area.
<aside> define the content of sidebar.
<bdi> allow you to set a text which can detach the location of parent text.
<command> define command button.
<details> described details of some parts of the document.
<dialog> define dialog box.
<summary> contains the title of detalis elements.
<figure> Provides independent streaming content (images, charts, photos, code, etc.).
<figcaption> define the title of <figure>
<footer> define the footer of section or document.
<header> define the header area of the document.
<mark> define text with tokens.
<meter> define the weights and measures. Only used for measures of known maximum and minimum values.
<nav> define the part of the navigation link.
<progress> define the progress of any type of task.
<ruby> define ruby comment (Chinese phonetic or character).
<rt> definition of characters (Chinese phonetic or character) interpretation or pronunciation.
<rp> Used in Ruby annotations to define what browsers do not support Ruby elements.
<section> define the sectionin the document.
<time> define date or time.
<wbr> specify where in the text is appropriate for adding newline characters.
